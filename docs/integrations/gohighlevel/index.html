GoHighLevel Integration

1. Overview​

1.1 Description​

GoHighLevel (GHL) Integration connects GoHighLevel (all-in-one marketing and CRM platform) with Newo Platform.

It enables:

  • Checking real-time calendar availability with dynamic calendar selection (by name, duration, or fixed)
  • Booking, cancelling, and rescheduling appointments in GHL calendars
  • Automatic contact creation and lookup in GHL CRM
  • Conversation sync with call status, recording URLs, and transcripts
  • Opportunity/deal creation and AI-powered pipeline stage management
  • Client history tracking with conversation notes
  • Existing client detection with appointment history retrieval
  • OAuth 2.0 and Private Token authentication

This integration is designed for marketing agencies, sales teams, and service businesses who need automated appointment scheduling, CRM synchronization, and conversation tracking through a conversational AI agent (voice or chat).

1.2 Use Cases​

  • When a client asks about available slots → Check GHL calendar availability with dynamic calendar selection
  • When a client wants to book an appointment → Find or create contact, book in selected calendar
  • When a client wants to cancel → Update appointment status to "cancelled" in GHL
  • When a client wants to reschedule → Update appointment start time in GHL
  • When a conversation ends → Sync transcript, summary, and recording to GHL Conversations
  • When a conversation ends → Create/update opportunity with AI-selected pipeline stage
  • When a conversation ends → Save conversation notes to contact profile
  • When a caller's phone is identified → Look up existing contact and upcoming appointments

2. Requirements​

Before installation:

  • Active GoHighLevel account
  • Either OAuth 2.0 authorization (Marketplace) or Private Integration Token
  • GHL Location ID (sub-account identifier)

3. How to Setup​

3.1 Step 1 — Authorize in GoHighLevel​

OAuth (Recommended — Marketplace):​

  1. In Newo platform, navigate to GHL integration settings
  2. Click the GoHighLevel authorization link
  3. Log in to your GHL account and grant required scopes
  4. Copy the authorization code from the redirect
  5. Paste the code into the ghl_oauth_code attribute

Private Token (Manual):​

  1. Log in to GoHighLevel → Settings → Integrations
  2. Create a Private Integration
  3. Copy the generated API Token
  4. Set ghl_auth_type to "Token"
  5. Paste the token into ghl_oauth_token attribute

3.2 Step 2 — Connect in Platform​

  1. Open Newo → Projects
  2. Set the following attributes in Builder / Attributes:

  2. On publish, the SetupFlow automatically:

    • Fetches custom values and pipeline definitions
    • Injects booking and availability payload schemas for the agent

This section explains how the integration works, how to configure automation, and how to test it.

4.1 How the Integration Works​

The integration combines scheduling (Calendars/Appointments), CRM (Contacts, Opportunities), and communication tracking (Conversations) capabilities via the event-driven system.

Available Actions​

  • Get Free Slots — Check calendar availability (GET /calendars/{id}/free-slots)
  • Create Appointment — Book in GHL calendar (POST /calendars/events/appointments)
  • Update Appointment — Cancel or reschedule (PUT /calendars/events/appointments/{id})
  • Search Contact — Find by phone (POST /contacts/search)
  • Create Contact — New contact with name, phone, email (POST /contacts/)
  • Get Appointments — Retrieve contact's future bookings (GET /contacts/{id}/appointments)
  • Upsert Opportunity — Create or update deal (POST /opportunities/upsert)
  • Search Conversations — Find existing thread (GET /conversations/search)
  • Create Conversation — New conversation thread (POST /conversations/)
  • Write Message — Log call/transcript (POST /conversations/messages/outbound)
  • Add Note — Attach summary to contact (POST /contacts/{id}/notes)
  • Get Calendars — Discover available calendars (GET /calendars/)
  • Get Pipelines — Fetch pipeline definitions (GET /opportunities/pipelines)

4.3 Where to Test​

Testing can be done through the Newo conversation interface (chat or voice channel). The AvailabilityFlow has a dedicated test_integration webhook endpoint for connectivity validation.

4.4 Testing and Verification​

To test the integration:​

  1. Setup: Publish and verify logs show calendar fetch, pipeline fetch, and OAuth/token success
  2. Existing Client: Start a conversation from a known phone — verify client context injected
  3. Availability: Ask about available slots — verify calendar selection mode works correctly
  4. Booking: Complete a booking — verify appointment in GHL calendar and contact created
  5. Cancellation: Cancel an appointment — verify status changed to "cancelled" in GHL
  6. Reschedule: Move an appointment — verify new time in GHL
  7. Post-Conversation: After session ends, verify:
    • Conversation message with transcript in GHL Conversations
    • Note added to contact profile
    • Opportunity stage updated in pipeline

If no action occurs:​

  • Ensure OAuth code or private token is set correctly
  • Verify ghl_location_id is correct
  • Confirm the integration was re-published after changes
  • Check that calendars were fetched during setup (verify ghl_calendar attribute)
  • Verify feature flags are enabled for desired functionality
  • For calendar selection by name/duration: check ghl_selection_criteria instructions

Note: This integration performs real operations in GoHighLevel. Appointments, contacts, conversations, and opportunities created through the agent are reflected in the live GHL system.

5. FAQ​

Q: Does the integration import historical data? A: No. Only events and records created after activation are managed. Existing client detection retrieves future appointments for context only.

Q: Can I connect multiple GHL locations? A: Each integration instance supports one Location ID. For multiple locations, create separate integration instances.

Q: What is the difference between OAuth and Token authentication? A: OAuth is recommended for Marketplace installations — it uses the standard authorization flow with scopes. Token uses a Private Integration Token generated manually in GHL settings, suitable for simpler setups.

Q: How does dynamic calendar selection work? A: Three modes: Single always uses the pre-selected calendar. Name uses an LLM to extract the service name from the conversation and match it to a calendar. Duration uses an LLM to extract the meeting length and match it to a calendar with that duration.

Q: How is conversation data synced to GHL? A: When ghl_sync_conversation is enabled, the integration creates an outbound message in GHL Conversations containing the full transcript, summary, channel info, recording URLs, and call status (completed, voicemail, no-answer, etc.).

Q: How does the AI select the pipeline stage? A: An LLM (Gemini) analyzes the conversation summary, category, business outcome, and transcript. It applies custom rules from ghl_pipeline_criteria and selects the most appropriate stage from ghl_pipeline_stages. The decision includes an explanation.

Q: What happens if a contact already exists? A: The integration searches by phone number. If found, the existing contact is used. If not, a new contact is created with name, phone, and email.

Q: Can I detect the caller's timezone from the conversation? A: Yes. Set ghl_use_timezone_from_conversation to True. The integration will extract timezone from the conversation context and use it for appointment scheduling. Otherwise, the business timezone is used.
