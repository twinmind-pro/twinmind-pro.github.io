Deliverect Integration Guide

1. What This Integration Does​

The Newo AI Agent integrates with Deliverect so your AI can act as a restaurant ordering assistant on top of a Deliverect-connected menu and channel link.

Once connected, the AI agent can:

  • answer menu questions from the synchronized Deliverect menu snapshot
  • create new pickup, delivery, eat-in, or curbside orders
  • enforce menu grounding by using only synced Deliverect items, PLUs, prices, and availability flags
  • store orders created through the agent so the same caller can later ask about them
  • request cancellation of an existing order when the order is still eligible
  • publish Deliverect ordering, status, and cancellation scenarios into the Newo Canvas

In plain terms: Deliverect remains the restaurant menu and order system. Newo becomes the conversational front desk that can take orders, check status, and request cancellations without asking the customer to open a web ordering link.

Figure S01. The Module - DeliverectIntegration settings block in Newo Builder.

2. Before You Start​

Make sure you have:

  • an active Deliverect partner/channel setup for the restaurant location
  • Deliverect Client ID and Client Secret
  • Deliverect Channel Name
  • Deliverect Channel Link ID for the restaurant location this agent will serve
  • Deliverect Account ID for the customer account that owns the location
  • a published menu for that Deliverect store/channel
  • access to the Newo project builder and permission to click Publish All

Production also requires Deliverect channel certification for the capabilities you plan to use, such as order creation, status lookup, and cancellation.

Deliverect credentials are partner-gated. They are issued by Deliverect during channel onboarding; there is no public self-service setup where a restaurant operator can generate every credential alone.

Deliverect values you need​

ValueWhat it is used for
Client IDOAuth machine-to-machine token request.
Client SecretOAuth machine-to-machine token request.
Channel NameChannel API order URL path, usually newoai.
Channel Link IDThe restaurant/channel location where orders are submitted.
Account IDCommerce API menu retrieval.
Location IDOptional store ID for menu retrieval. Leave blank unless Deliverect gave a separate value.

Where to find these values in Deliverect​

Most Deliverect identifiers come from the partner/channel setup, not from the restaurant-facing menu editor.

ValueWhere to get it
Client ID and Client SecretDeliverect channel onboarding email or partner developer credentials page. These belong to your channel app, not to an individual restaurant.
Channel NameDeliverect channel onboarding details. Use the exact value assigned to the channel; it is case-sensitive.
Channel Link IDDeliverect partner dashboard -> customer account -> location/store -> channel links. Copy the link for the exact restaurant location this agent should submit orders to.
Account IDDeliverect partner dashboard -> Admin. Use the account that owns the Channel Link ID.
Location IDLocation/store details in Deliverect. Leave this blank unless Deliverect provides a separate store ID for menu pulls.

If you are unsure which ID is which, ask Deliverect for the values for: client credentials, channel name, channelLinkId, accountId, and optional store/location ID for the same restaurant location.

Figure S02. Deliverect onboarding credentials or partner dashboard values, split across two screenshots.

Wrong Channel Link ID = wrong restaurant. Channel Link IDs are location-specific. Double-check this before sending any test order.

3. Connecting for the First Time​

3.1 Open the Newo Project​

  1. Open Newo Portal.
  2. Open the projects list.
  3. Select your project.

Figure S03. Opening the Deliverect project in Newo Builder.

3.2 Open Deliverect Settings​

  1. Inside the project, open Builder.
  2. Click Attributes in the left sidebar.
  3. Find Module - DeliverectIntegration.
  4. Open the group.

3.3 Paste OAuth Credentials​

Paste and save:

  1. 01. Client ID (deliverect_client_id)
  2. 02. Client Secret (deliverect_client_secret)

Leave 03. API Base URL on the staging default unless Deliverect has already certified the channel for production.

Figure S04. Client ID and Client Secret fields in Newo.

3.4 Paste Channel and Account Values​

Paste and save:

  1. 04. Channel Name (deliverect_channel_name), usually newoai
  2. 05. Channel Link ID (deliverect_channel_link_id)
  3. 06. Account ID (deliverect_account_id)
  4. 07. Location ID (deliverect_location_id), optional

If Location ID is blank, the integration uses Channel Link ID as the store ID for menu retrieval.

Figure S05. Channel Name, Channel Link ID, Account ID, and optional Location ID.

3.5 Review Feature Switches​

For a normal deployment, leave these enabled:

  • Enable Order Creation (deliverect_enable_ordering)
  • Enable Order Cancellation (deliverect_enable_cancellation)
  • Publish Default Canvas (deliverect_setup_scenarios)

Turn a feature off only if the restaurant or Deliverect certification scope does not allow it yet.

Figure S06. Deliverect feature switches.

3.6 Save and Publish​

Important: save each edited attribute. After editing a setting in Newo, click Save for that setting before moving on.

When all fields are saved:

  1. Click Publish All.
  2. Wait for setup to finish.
  3. Confirm the menu cache and Canvas items are created.

Figure S07. Publish All after saving Deliverect settings.

What happens after Publish All:

  • Newo creates the HTTP connector deliverect_connector.
  • Newo creates a hidden setup persona used for background setup actions.
  • The integration registers custom Deliverect tools.
  • The integration disables generic availability, booking, and cancellation tools that do not apply to restaurant ordering.
  • The integration requests a Deliverect OAuth token when needed and caches it.
  • The integration pulls the Deliverect menu from the Commerce API.
  • The menu is written into prompt-facing restaurant menu/RAG attributes.
  • If Publish Default Canvas is on, Deliverect intents and scenarios are added to the Canvas.

3.7 Verify Setup​

After publishing, check:

  • The Deliverect settings group has saved credentials and channel IDs.
  • deliverect_access_token_expires_at is set to a future timestamp after the first API call.
  • The synchronized menu appears in project_attributes_restaurant_menu or the RAG menu attribute.
  • The Canvas contains Deliverect ordering and cancellation scenarios.
  • A test chat asking "What's on the menu?" returns real items from the Deliverect menu.

Figure S08. Deliverect Canvas scenarios and intent types after publish.

Whenever you change credentials, channel IDs, feature switches, or production/staging URL:

  1. Save the changed setting.
  2. Click Publish All.
  3. Run at least one menu or ordering test.

5. What the AI Agent Can Do​

5.1 Answer Menu Questions​

The agent answers menu questions from the synchronized Deliverect menu snapshot. The menu is pulled from:

GET /commerce/{accountId}/stores/{storeId}/menus

The integration stores:

  • item names
  • PLUs
  • prices in minor currency units
  • descriptions
  • categories
  • required and optional modifier groups
  • availability flags such as hidden or snoozed

You will see in Deliverect: no order is created when the customer only asks menu questions.

5.2 Place an Order​

The agent can submit a new order through:

POST /{channelName}/order/{channelLinkId}

What happens automatically:

  • The agent extracts customer name, phone, fulfillment type, delivery address when needed, requested time, and confirmed items.
  • The integration maps item names to synced menu PLUs and prices.
  • The integration blocks submission if the customer selected an item that is missing, snoozed, or not in the current menu.
  • The integration blocks delivery orders if the delivery address is missing.
  • The integration generates a channelOrderId and channelOrderDisplayId.
  • The integration submits the order with payment metadata where the order is not already paid.
  • On success, the order is stored in the persona deliverect_orders attribute.

You will see in Deliverect: a new channel order for the configured Channel Link ID.

Figure S11. New order in Deliverect after the agent submits it.

5.3 Request Cancellation​

The agent can request cancellation by sending a cancellation status payload to:

POST /{channelName}/order/{channelLinkId}

The payload includes:

  • channelOrderId
  • channelOrderDisplayId
  • status: 100
  • cancellation reason

The flow will not send an automatic cancellation request if the locally known Deliverect status is already 20 or higher, because the order has advanced far enough that a human or restaurant-side process should handle it.

You will see in Deliverect: a cancellation request. Final cancellation depends on restaurant/POS acceptance.

5.4 What the Agent Cannot Do​

The current integration does not:

  • edit menus in Deliverect
  • create or manage appointments
  • check calendar availability
  • process payments inside Newo
  • support multiple Channel Link IDs in one Newo project
  • guarantee cancellation after a restaurant or POS has accepted the order

6. How to Test That Everything Works​

Before going live, run through this checklist on staging or a quiet test location.

6.1 After Setup​

After clicking Publish All, confirm:

  • Client ID and Client Secret are filled in.
  • API Base URL matches the credential environment.
  • Channel Name, Channel Link ID, and Account ID are filled in.
  • Enable Order Creation and Enable Order Cancellation match your certification scope.
  • The Canvas contains Deliverect order, status, and cancellation scenarios.
  • The menu cache contains real Deliverect items.

6.2 Test Menu Knowledge​

  1. Start a test chat or call session.
  2. Ask: "What's on the menu?"
  3. Confirm the agent lists real Deliverect items.
  4. Ask about an item that is not in the menu.
  5. Confirm the agent does not invent it and offers available alternatives.

6.3 Test Order Placement​

  1. Ask: "Can I place an order for pickup?"
  2. Choose one real menu item.
  3. Include any required modifiers.
  4. Provide first name, last name if your scenario asks for it, and phone number.
  5. Confirm the summary.
  6. Confirm the agent says the order-submission trigger phrase.
  7. Confirm a new order appears in Deliverect under the configured Channel Link ID.

6.4 Test Delivery Order Guard​

  1. Ask for delivery.
  2. Do not provide an address at first.
  3. Confirm the agent asks for the missing delivery address before submitting.
  4. Provide the address and complete the order.

6.5 Test Cancellation​

  1. Use a fresh test order in a cancellable state.
  2. Ask to cancel the order.
  3. Confirm the agent restates the target order and asks for confirmation.
  4. Confirm the agent says the cancellation trigger phrase.
  5. Confirm Deliverect receives the cancellation request.

7. Canvas Auto-Setup​

When Publish Default Canvas is on, the integration publishes Deliverect ordering content into the Canvas.

7.1 What Gets Added​

IntentScenario
[T] Food Order Request via AgentScenario 1: "Creating Food Order via Agent"
[T] Food Order Status RequestScenario 2: "Checking Delivery Order Status via Agent"
[T] Food Order Cancellation RequestScenario 3: "Canceling Deliverect Order via Agent"

These are starting templates. Review them and adapt the wording to the restaurant, but keep the trigger phrases intact.

7.2 Trigger Phrases​

Each scenario contains a phrase the agent must say before the matching custom tool fires.

Trigger phraseFires
"I will submit your order to the restaurant right now. Please give me a moment, and I'll get back to you shortly."Order creation
"Thanks. I'll continue with your order now."Order continuation after missing data
"I will ask the restaurant to cancel that order right now. Please give me a moment."Cancellation request

Do not reword, translate, or remove trigger phrases. If a trigger phrase changes, the corresponding action may stop firing.

7.3 Safe Canvas Customization​

Safe changes:

  • Reword the greeting, item questions, and summary.
  • Add upsell prompts before the final order summary.
  • Add restaurant-specific disclaimers about cancellation windows.
  • Change how the agent explains status results.

Unsafe changes:

  • Removing the menu-grounding rule.
  • Letting the agent submit orders before final customer confirmation.
  • Rewording trigger phrases.
  • Saying cancellation is guaranteed before Deliverect confirms the result.

8. Reconnecting, Rotating Credentials, and Production Cutover​

8.1 When to Reconnect​

Reconnect or republish when:

  • Deliverect rotates the Client Secret.
  • You switch from staging to production.
  • The Channel Link ID changes.
  • Menu questions return an empty or outdated menu.
  • Order submission starts failing with authentication errors.

8.2 Rotate Client Secret​

  1. Get the new Client Secret from Deliverect.
  2. Paste it into 02. Client Secret.
  3. Save the setting.
  4. Click Publish All.
  5. Run a menu test and a small order test.

8.3 Switch to Production​

After Deliverect certification:

  1. Replace staging Client ID and Client Secret with production values.
  2. Set 03. API Base URL to the production URL Deliverect provided, usually https://api.deliverect.com.
  3. Set Token Audience to the matching production audience if Deliverect requires it.
  4. Save each changed setting.
  5. Click Publish All.
  6. Run order placement, status, and cancellation tests.

9. Troubleshooting​

The agent cannot answer menu questions​

Likely cause: the menu cache is empty or the Account ID / store ID is wrong.

Fix:

  1. Confirm Account ID belongs to the same Deliverect account as the Channel Link ID.
  2. Confirm the restaurant has a published menu.
  3. Confirm Location ID is blank unless Deliverect gave a separate store ID.
  4. Click Publish All to pull the menu again.

Publish All or the first API call fails with authentication​

Likely cause: Client ID, Client Secret, Token Audience, or API Base URL do not match.

Fix:

  1. Check for leading/trailing spaces in Client ID and Client Secret.
  2. Confirm staging credentials use https://api.staging.deliverect.com.
  3. Confirm production credentials use the production API base URL.
  4. Save and publish again.

Orders appear in the wrong restaurant​

Likely cause: Channel Link ID points to a different restaurant location.

Fix:

  1. Copy the correct Channel Link ID from Deliverect.
  2. Paste it into 05. Channel Link ID.
  3. Save and publish.
  4. Place a small test order and verify the location in Deliverect.

The agent says an item cannot be submitted​

Likely cause: the item is missing from deliverect_menu_state, has no matching PLU, or is snoozed/unavailable.

Fix:

  1. Confirm the item is available in Deliverect.
  2. Republish to refresh the menu snapshot.
  3. Ask the customer to choose from the synced menu names.

Cancellation is not sent automatically​

Likely cause: cancellation is disabled, the order target is unclear, another cancellation is already in progress, or the latest known status is already 20 or higher.

Fix:

  1. Confirm Enable Order Cancellation is on.
  2. Confirm the exact order with the customer.
  3. If the order has advanced, hand off to the restaurant or a human team member.

The agent ignores ordering/cancellation language​

Likely cause: Canvas scenario trigger phrases were edited or the default Canvas was not published.

Fix:

  1. Confirm Publish Default Canvas is on.
  2. Confirm the Deliverect intents and scenarios exist.
  3. Restore the trigger phrases from Section 7.2.
  4. Click Publish All.

10. FAQ​

Does this create real orders in Deliverect?
Yes. Order creation submits a real channel order to the configured Channel Link ID.

Does the agent use a live menu on every turn?
No. The integration pulls a menu snapshot and injects it into prompt/RAG attributes. Republish or refresh setup to pull a newer menu.

Can one Newo project serve multiple restaurants?
No. One project is configured for one Channel Link ID. Use separate projects for separate restaurant locations.

Can the agent take payment?
No. The integration sends order data to Deliverect. Payment remains handled by Deliverect, the POS, or the restaurant's normal process.

Can the agent cancel any order?
No. It sends cancellation requests only when the order is clearly identified and still eligible. Final cancellation depends on Deliverect/POS/restaurant acceptance.

What order types are supported?
The extraction supports pickup, delivery, eat-in, and curbside. Pickup is the safest default unless the customer clearly asks for another fulfillment type.

Why does the agent refuse an item the customer asked for?
The agent must use the synchronized menu. If an item is missing, snoozed, or has no matching PLU, the integration blocks submission and asks the customer to choose an available item.

Do I need webhook URLs for the basic flow?
No. The standard setup uses API calls for menu pull, order creation, status lookup, and cancellation request. Webhook-driven operational state can be handled separately with Deliverect onboarding if needed.

11. Support Handoff Notes​

When handing off to support or an implementation partner, include:

  • Deliverect environment: staging or production
  • API Base URL
  • Channel Name
  • Channel Link ID
  • Account ID
  • optional Location ID
  • which features are enabled
  • whether default Canvas setup was used
  • whether the latest menu cache contains the expected items

Key implementation details:

  • OAuth token request: POST /oauth/token
  • Menu pull: GET /commerce/{accountId}/stores/{storeId}/menus
  • Order creation: POST /{channelName}/order/{channelLinkId}
  • Cancellation request: POST /{channelName}/order/{channelLinkId} with status: 100
  • Stored user orders: persona attribute deliverect_orders
  • Prompt menu text: project_attributes_restaurant_menu and project_attributes_rag_knowledge_base_append_text
  • Structured menu cache: deliverect_menu_state

The two most common fixes are:

  1. Re-save credentials/channel IDs and click Publish All.
  2. Verify the Channel Link ID and Account ID belong to the same Deliverect location/account.
