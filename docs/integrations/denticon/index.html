Denticon Integration Guide

1. What This Integration Does​

The Newo AI Agent integrates with Denticon so your AI can help patients schedule and manage dental appointments for your practice.

Once connected, the AI agent can:

  • check appointment availability in Denticon
  • book appointments in Denticon
  • cancel existing appointments
  • recognize existing patients and find their upcoming visits
  • ask the patient which office they want when multiple offices are available

In plain terms: the AI handles the conversation, Denticon remains the scheduling system, and the patient is guided to the correct office during the conversation.

Figure S01. Newo Projects page.

2. Create the Denticon Project​

If the Denticon project has not been created yet in Newo, create it first.

  1. Open Newo -> Projects.
  2. Click Create New Project.
  3. Enter a project identifier and title for the Denticon integration.
  4. In Module, select denticon_integration.
  5. Leave the module version at the default value unless your implementation partner tells you to use a different one.
  6. Click Create.
  7. When the project appears in the list, click Manage to continue.

3. Before You Start​

Make sure you have:

  • an active Denticon account with API access
  • access to the Newo project builder for your project
  • permission to edit project settings and publish

You will need these Denticon credentials:

  • Client ID
  • Client Secret
  • Subscription Key

You can find these values in the PDDS API Credentials Portal:

  1. Open Client List.
  2. Find the Denticon integration client you need.
  3. Click View Details.
  4. Copy the Client ID, Client Secret, and Subscription Key from the details page.

Important: The Denticon Base URL is already prefilled in the integration. For a standard setup, you only need to enter the three credentials above.

Figure S05. In the PDDS API Credentials Portal, open the correct Denticon client from the Client List.

Figure S06. Copy the Client ID, Client Secret, and Subscription Key from the client details page.

4. Connecting for the First Time​

  1. Open your project in Newo and go to Builder -> Attributes.
  2. Find the Denticon section.
  3. Enter your Client ID.
  4. Enter your Client Secret.
  5. Enter your Subscription Key.
  6. Click Save.
  7. Click Publish All.

After that, the integration is ready to use.

5. Settings Reference​

You can find the setup fields in Newo Project -> Builder -> Attributes -> Denticon.

SettingWhat it doesRequired
Client IDDenticon API client identifierYes
Client SecretDenticon API client secretYes
Subscription KeyDenticon API subscription keyYes

Recommended setup: leave the default Denticon settings as they are and only fill in the three required credentials.

6. Multi-Location Behavior​

Denticon is configured for multi-location behavior by default.

This means:

  • the user does not pre-select an office during the initial setup
  • the agent must ask the patient which office or location they want during the conversation
  • this office-questioning behavior must be included in the agent scenarios

Important: If the patient has not specified a location yet, the agent should ask for the preferred office before continuing with availability or booking.

7. Agent Scenario Requirement​

The agent scenarios must explicitly enforce office clarification before scheduling actions.

In practice, this means:

  • before checking availability, the agent should ask which office the patient wants if it is still unclear
  • before booking an appointment, the agent should confirm the office if it has not already been established
  • the conversation should not move into Denticon availability or booking flows without a clear office choice

8. After Setup​

Once Publish All is complete, the integration can be used in live conversations.

At that point:

  • the agent can start handling Denticon scheduling requests
  • the patient will be asked for the preferred office during the conversation when needed
  • the setup is complete from the user's side
