HouseCallPro Integration

1. Overview​

1.1 Description​

HouseCallPro Integration connects HouseCallPro (field service management platform) with Newo Platform.

It enables:

  • Checking available appointment slots with intelligent employee selection (by role, job type, or zip code)
  • Booking appointments with automatic customer search, creation, and job management
  • Cancelling existing appointments
  • Recognizing returning customers on inbound calls with upcoming job context
  • Caching company data (employees, job types, locations) with configurable refresh intervals
  • Automatic line item matching using AI for job cost estimation

This integration is designed for home service businesses (HVAC, plumbing, electrical, cleaning, etc.) who need automated appointment scheduling through a conversational AI agent (voice or chat).

1.2 Use Cases​

  • When a customer asks about available slots -> AI selects the right employee and returns open time slots
  • When a customer wants to book a service -> Search or create customer record, create job with line items, and schedule appointment
  • When a customer wants to cancel -> Cancel the appointment via HouseCallPro API
  • When an inbound call arrives -> Automatically look up customer by phone and display upcoming jobs to the agent
  • When a customer describes a service need -> AI matches the description to job types and line items for accurate quoting
  • When a session starts -> Static data (employees, job types, company info) is refreshed if cache is stale

2. Requirements​

Before installation:

  • Active HouseCallPro account (Pro plan or higher for API access)
  • HouseCallPro API key for authentication
  • At least one employee configured with appropriate permissions
  • For POC method: at least one employee marked as Point of Contact in HouseCallPro

3. How to Setup​

3.1 Step 1 — Get API Credentials​

  1. Log in to your HouseCallPro account
  2. Navigate to account settings or developer/API section
  3. Generate or locate your API key
  4. Note your company timezone setting

3.2 Step 2 — Connect in Platform​

  1. Open Newo -> Projects
  2. Set the following attributes in Builder / Attributes:
AttributeRequiredDescription
hcp_api_keyYesHouseCallPro API key for authentication
hcp_base_urlNoAPI base URL (default: https://api.housecallpro.com/)
hcp_enable_slot_checkNoEnable availability checks (default: True)
hcp_enable_bookingNoEnable appointment creation (default: True)
hcp_enable_cancellationNoEnable cancellations (default: True)
hcp_auto_employee_typeNoEmployee assignment method: poc, job_type, or zip_code (default: poc)
hcp_booking_durationNoAppointment duration in hours (default: 2)
hcp_show_for_daysNoDays to search for availability (default: 5)
hcp_availability_offsetNoDays before first available slot (default: 2)
hcp_check_existing_clientNoLook up customer on inbound call (default: True)
hcp_auto_add_line_itemsNoAuto-match line items to jobs via AI (default: False)
hcp_line_itemsNoService catalog as JSON array of items
hcp_job_type_employee_selection_promptNoCustom LLM prompt for job-type employee matching
hcp_static_data_update_intervalNoCache refresh interval in seconds (default: 86400)
  1. Click Save + Publish All
  2. On publish, the SetupFlow automatically:
    • Creates an HTTP connector for all API operations
    • Fetches and caches company info, employees, and job types
    • Registers availability, booking, and cancellation tools with the agent
    • Injects payload schemas for the ConvoAgent tool calling framework

4. How to Use​

This section explains how the integration works, how to configure automation, and how to test it.

4.1 How the Integration Works​

The integration uses HouseCallPro's REST API for job management, appointment scheduling, customer records, and employee data. A key architectural feature is the two-level booking model: a Job is created first (representing the service request), then an Appointment is attached to it (representing the scheduled time slot). Employee selection supports three AI-powered strategies. Static data is cached with a configurable TTL to minimize API calls.

  • A Trigger is a system event that starts a flow
  • An Action is the API operation performed in HouseCallPro

Available Triggers​

Session Endend_sessionClears temporary customer data

Available Actions​

  • Get Company — Fetch company info and locations (GET /company)
  • Get Job Types — Fetch available service types (GET /job_fields/job_types)
  • Get Employees — Fetch employees, optionally by location (GET /employees)
  • Get Booking Windows — Fetch available time slots (GET /company/schedule_availability/booking_windows)
  • Get Jobs — Fetch scheduled jobs for conflict detection (GET /jobs)
  • Get Events — Fetch calendar events for conflict detection (GET /events)
  • Search Customer — Find existing customer by phone (GET /customers)
  • Create Customer — Create a new customer record (POST /customers)
  • Create Job — Create a service job (POST /jobs)
  • Create Appointment — Schedule a time slot on a job (POST /jobs/{id}/appointments)
  • Cancel Appointment — Delete a scheduled appointment (DELETE /jobs/{id}/appointments/{apt_id})
  • Get Job Appointments — Fetch appointments for a job (GET /jobs/{id}/appointments)

4.3 Where to Test​

Testing can be done through the Newo conversation interface (voice or chat). Each flow has a dedicated webhook test endpoint:

  • availability_test_event — AvailabilityFlow
  • appointment_test_event — AppointmentFlow
  • appointment_test_cancel_event — CancellationFlow
  • get_client_test_event — ExistingClientFlow
  • update_static_test_event — StaticDataFlow

4.4 Testing and Verification​

To test the integration:​

  1. Setup: Publish the project and verify static data is cached (check hcp_company_employees is populated)
  2. Static Data: Verify hcp_company_info, hcp_job_types, and hcp_time_zone are populated
  3. Existing Client: Call from a known customer's phone — verify agent greets with context
  4. Availability: Ask "What slots are available this week?" — verify slots returned with correct times
  5. Booking: Complete a booking conversation with address — verify job and appointment appear in HouseCallPro
  6. Cancellation: Request cancellation — verify appointment removed from HouseCallPro

If no action occurs:​

  • Ensure hcp_api_key is set correctly and has appropriate permissions
  • Verify static data was cached (check hcp_company_employees is not empty)
  • Confirm the relevant feature flags are enabled (hcp_enable_booking, hcp_enable_slot_check, hcp_enable_cancellation)
  • For POC method: verify at least one employee has Point of Contact permission enabled in HouseCallPro
  • For job_type method: verify employees have relevant tags/skills configured
  • For zip_code method: verify employees have zip code tags matching the service area
  • Check that hcp_booking_duration and hcp_show_for_days are reasonable values
  • Re-publish the project if the API key was changed

Note: This integration performs real operations in HouseCallPro. Jobs, appointments, and customers created or cancelled through the agent are reflected in the live HouseCallPro system.

5. FAQ​

Q: What are the three employee selection methods and when should I use each? A: POC (default) assigns the Point of Contact employee — best for single-technician businesses or when one person handles all dispatch. Job Type uses AI to match the customer's service description to employee skills/tags — best for multi-specialty teams. Zip Code matches by service area — best for geographically distributed teams. Set via hcp_auto_employee_type.

Q: How does the AI line item matching work? A: When hcp_auto_add_line_items is enabled, the integration uses Gemini 2.5 Flash to match the customer's service description against the hcp_line_items catalog and hcp_job_types. The AI selects the most relevant items and attaches them to the job for accurate quoting.

Q: What customer information is required for booking? A: First name, phone number (E.164 format), and full service address (street, city, state, zip, country) are required. Last name defaults to the first name if not provided. Email is optional.

Q: How often is static data refreshed? A: By default every 24 hours (hcp_static_data_update_interval = 86400 seconds). The cache is checked on each session start. You can trigger a manual refresh via the hcp_update_static_data event or by re-publishing the project.

Q: What is the default appointment duration? A: 2 hours, configurable via hcp_booking_duration. The end time is calculated as start time + duration in hours. Times are converted from the business timezone to UTC for the API.

Q: How does existing customer recognition work? A: When hcp_check_existing_client is enabled, the integration looks up the caller's phone number on inbound calls. If a customer is found, their profile and upcoming jobs are injected into the agent's context as ExistingCustomerInfo, allowing the agent to greet them by name and reference their scheduled services.

Q: What happens if no employees match the selected method? A: The integration falls back to the POC method. If no POC employee is found either, the availability check will return no slots and the agent will inform the customer accordingly.
