Exely Integration Guide

See also: TESTING_GUIDE.md

1. What This Integration Does​

The Newo AI Agent integrates with Exely (hotel PMS / channel manager) so your AI can act as a hotel reservation assistant for your property.

Once connected, the AI agent can:

  • check room availability for requested stay dates and guest counts
  • present room options with pricing and meal plans
  • create hotel reservations (two-step verify-then-create flow)
  • cancel reservations with penalty calculation
  • modify/reschedule reservations (date changes, room upgrades)
  • look up existing reservations by confirmation number
  • auto-add default hotel booking scenarios to Canvas

In plain terms: the AI can talk to guests about room availability and bookings while Exely acts as the hotel reservation backend.

Manager Quick View​

QuestionShort answer
What business problem does this solve?Lets the AI handle hotel availability, booking, cancellation, modification, and booking lookup without a human agent doing those actions manually.
Who is this for?Hotels and hospitality businesses using Exely (formerly Bnovo) as their property management system.
What is needed before setup?Exely API credentials (Client ID + Client Secret) and the Property ID.
How long does setup usually take?5-10 minutes if API connections are already enabled in the Exely extranet.
How do we know it worked?Publish All completes successfully and exely_access_token is populated.

Best Fit / Not a Fit​

Best fit:

  • hotel reservation operations via Exely
  • AI-led guest conversations about room availability and bookings
  • teams that want one Newo project per hotel/property

Not a fit:

  • CRM-heavy workflows or lead pipeline automation
  • guest recognition based only on phone number
  • multi-property routing inside one Exely setup
  • payment processing (not yet supported)

2. Before You Start​

Make sure you have:

  • access to the Newo project builder for your project
  • permission to edit project settings and publish
  • API connections section enabled in the Exely extranet (may need to request from Exely support)
  • Exely API credentials (Client ID and Client Secret)
  • the exact Property ID for your hotel

Who Usually Provides What​

RoleUsually provides
Hotel / client teamExely account access, Property ID, operational testing help
Exely supportAPI connections enablement, Client ID, Client Secret
Newo implementer / PMNewo setup, feature toggles, publish, test plan, go-live validation

You will need these values before setup:

  • Client ID (OAuth2)
  • Client Secret (OAuth2)
  • Property ID
  • Base URL (default: https://connect.hopenapi.com)
  • Language Code (default: en)

Note: The API connections section is not enabled by default in the Exely extranet. You must contact Exely support (welcome@exely.com) to enable it before you can generate API credentials.

3. Connecting for the First Time​

3.1 Enable API Access in Exely​

  1. Contact your Exely account manager or email welcome@exely.com
  2. Request to enable the "API connections" section in your extranet
  3. Wait for confirmation (typically 1-2 business days)

3.2 Create an API Connection​

  1. Log in to the Exely extranet at https://secure.exely.com/app/

  2. Navigate to Property Settings → API Connections

Figure 1. Navigate to API Connections in the Property Settings menu.

  1. Click "Create a connection"

  2. Enter a name (e.g., Newo AI Integration) and select the required APIs:

    • Content API
    • Search API
    • Reservation API
    • Read Reservation API
    • PMS Universal API v2

Figure 2. Enter a connection name and select the required APIs.

  1. Click "Save and enable access"

Figure 3. Choose "Save and enable access" to activate the connection immediately.

  1. Copy the generated Client ID and Client Secret

Figure 4. Copy the Client ID and Client Secret — you will need them in the Newo platform setup.

Your created connections will appear in the list:

Figure 5. List of created API connections.

You can manage connections (edit, enable/disable, delete) using the action menu:

Figure 6. Connection management actions.

Tip: Your Property ID can be found in the extranet URL: https://secure.exely.com/app/Property/{PROPERTY_ID}/... — or ask Exely support for it.

3.3 Configure Exely in Newo​

  1. Open your project in Newo.
  2. Go to Builder → Attributes.
  3. Find the 12. Exely Settings section.
  4. Fill in:
    • 01. Base URL — https://connect.hopenapi.com (production) or https://connect.test.hopenapi.com (testing)
    • 02. Property ID — your hotel's property identifier
    • 03. Language Code — language for room descriptions (en, ru, de, etc.)
    • 04. Client ID — OAuth2 Client ID from step 3.2
    • 05. Client Secret — OAuth2 Client Secret from step 3.2
  5. Review the feature toggles (all enabled by default):
    • 07. Enable Availability Check
    • 08. Enable Booking
    • 09. Enable Cancellation
    • 10. Enable Reschedule
    • 11. Enable Booking Lookup
  6. Click Save.
  7. Click Publish All.

3.4 What Happens After Publish​

When you publish:

  • Newo validates the integration registry ID
  • creates or updates HTTP connectors:
    • exely_connector (main API)
    • exely_token_connector (OAuth2 authentication)
  • creates or refreshes the setup persona and actor
  • requests an OAuth2 access token from Exely (valid 15 minutes, auto-refreshed)
  • stores the token in exely_access_token
  • registers all 5 Exely custom tools (availability, booking, cancellation, modification, lookup)
  • auto-adds default hotel scenarios and intent types to Canvas

3.5 What "Successful Setup" Looks Like​

After setup is complete, you should expect all of the following:

  • exely_access_token has a non-empty value
  • the project can run an availability check without authentication errors
  • the Exely settings stay saved after refresh
  • the booking-related tools are available to the agent
  • hotel-specific scenarios and intent types appear in Canvas

5. What the AI Agent Can Do​

5.1 Check Availability​

The AI checks Exely availability for:

  • check-in date (arrival)
  • check-out date (departure)
  • number of adult guests

The integration calls:

  • GET /api/search/v1/properties/{propertyId}/room-stays
  • GET /api/content/v1/meal-plans

What the agent receives:

  • available room types with names
  • rate plans with names
  • meal plan information (Room Only, Bed & Breakfast, etc.)
  • total pricing per room type
  • checksums for booking verification

5.2 Create a Reservation​

The AI can create a reservation after it has:

  • stay dates (arrival + departure)
  • adult count
  • guest first name and last name
  • guest phone number
  • (optional) guest email

The integration uses a two-step process:

  1. POST /api/reservation/v1/bookings/verify — validates the booking and returns a createBookingToken
  2. POST /api/reservation/v1/bookings — creates the confirmed reservation using the token

On success, the agent:

  • stores booking metadata in the persona bookings attribute (NAF standard)
  • communicates the confirmation number, total price, and dates to the guest

Note: The createBookingToken is single-use and valid for 24 hours.

5.3 Cancel a Reservation​

The AI can cancel a reservation when the guest provides their confirmation number.

The integration follows a three-step process:

  1. GET /api/reservation/v1/bookings/{number} — retrieves booking details
  2. GET /api/reservation/v1/bookings/{number}/calculate-cancellation-penalty — calculates the penalty
  3. POST /api/reservation/v1/bookings/{number}/cancel — cancels with the expected penalty amount

The agent communicates any penalty to the guest before proceeding.

5.4 Modify / Reschedule a Reservation​

The AI can update an existing reservation by changing stay dates or room type.

The integration follows a three-step process:

  1. GET /api/reservation/v1/bookings/{number} — retrieves current booking (including version for optimistic locking)
  2. POST /api/reservation/v1/bookings/{number}/verify — verifies the modification is possible
  3. POST /api/reservation/v1/bookings/{number}/modify — applies the changes

Note: Modification requires the booking version field for optimistic locking. If a 409 conflict occurs, the booking is re-fetched and the modification retried.

5.5 Look Up an Existing Reservation​

The AI can retrieve booking details when the guest provides their confirmation number.

The integration calls:

  • GET /api/reservation/v1/bookings/{number}

The agent reads back: status, dates, room type, guest name, and total price.

5.6 Auto-Setup Canvas Scenarios​

If Canvas auto-setup is enabled, Exely inserts:

  • 5 hotel-specific scenarios: Room Booking, Cancellation, Modification, Booking Lookup, Room Availability
  • 5 corresponding intent types that trigger the scenarios
  • removes conflicting default appointment/reservation scenarios

This happens through:

  • CanvasSetupFlow
  • SetupCanvasSkill + SetupLibrarySkill

7. Technical Architecture​

7.1 Three-Layer Architecture​

Layer 1 — Entry Flows (Availability, Booking, Cancellation, Reschedule, ExistingClient)
    Receive ConvoAgent events, extract parameters via LLM, dispatch to Layer 2

Layer 2 — ApiFlow
    Build API requests, chain multi-step operations, route results back to Layer 1

Layer 3 — NetworkFlow
    HTTP execution via exely_connector, OAuth2 token management via exely_token_connector

7.2 Trigger Events​

All operations use custom Newo tools with conversation history-based parameter extraction.

EventSourcePurpose
exely_check_availabilityCustom toolCheck room availability
exely_create_bookingCustom toolCreate reservation
exely_cancel_bookingCustom toolCancel reservation
exely_modify_bookingCustom toolModify/reschedule reservation
exely_lookup_bookingCustom toolLook up reservation details

7.3 Internal Events​

EventPurpose
exely_execute_requestSend authenticated HTTP request through NetworkFlow
exely_result_successRoute successful API response to ApiFlow
exely_result_errorRoute failed API response to ApiFlow
exely_check_availability_requestDispatch availability check to ApiFlow
exely_book_slot_requestDispatch booking to ApiFlow
exely_cancel_booking_requestDispatch cancellation to ApiFlow
exely_reschedule_requestDispatch modification to ApiFlow
exely_lookup_requestDispatch lookup to ApiFlow

7.4 Authentication​

  • Method: OAuth2 Client Credentials
  • Token endpoint: {base_url}/auth/token
  • Token lifetime: 15 minutes
  • Auto-refresh: On HTTP 401, token is refreshed and request retried (max 1 retry)
  • Rate limits: Auth endpoint: 3 req/s, 15 req/min, 300 req/hr per IP

7.5 High-Level Sequence​

8. How to Test That Everything Works​

Before going live, run through this checklist.

Recommended Test Order​

  1. Availability — proves authentication and property targeting work
  2. Booking — proves write access and two-step flow work
  3. Booking Lookup — proves existing reservation retrieval works
  4. Cancellation — proves post-booking lifecycle works
  5. Modification — proves date/room change flow works

8.1 After Setup​

After clicking Publish All, confirm:

  • exely_access_token is populated (non-empty)
  • no publish errors appear
  • exely_setup_persona_id exists
  • feature toggles have the intended values
  • hotel scenarios appear in Canvas

8.2 Test Availability​

  1. Start a test conversation.
  2. Ask: "Do you have any rooms available for May 1st to May 3rd for 2 guests?"
  3. Confirm the agent says "Let me check available rooms for you" and returns room options with prices.

8.3 Test Booking​

  1. Continue from availability or start a new conversation.
  2. Select a room type from the options.
  3. Provide guest details when asked (name, phone, email).
  4. Confirm the agent says "I'm booking your room now" and returns a confirmation number.

8.4 Test Booking Lookup​

  1. Start a new conversation.
  2. Ask: "Can you check my reservation? My booking number is [number]."
  3. Confirm the agent returns status, dates, room type, and total price.

8.5 Test Cancellation​

  1. Start a new conversation.
  2. Say: "I need to cancel my reservation. My booking number is [number], check-in [date]."
  3. Confirm the agent asks for confirmation, then cancels.

8.6 Test Modification​

  1. Start a new conversation.
  2. Say: "I need to change my booking [number] to new dates [dates]."
  3. Confirm the agent checks availability for new dates and offers to update.

9. FAQ and Troubleshooting​

Why does exely_access_token stay empty after publish?​

Check:

  • Client ID and Client Secret are correctly filled in
  • Base URL is correct (https://connect.hopenapi.com for production)
  • API connection is enabled and active in the Exely extranet

Why is availability working but booking fails?​

Check:

  • 08. Enable Booking is turned on
  • required guest fields are present in the conversation (name, phone)
  • the Exely API credentials have write permissions

Why can't the agent find my reservation?​

Most commonly:

  • the confirmation number format is wrong
  • the booking was made in a different property
  • 11. Enable Booking Lookup is turned off

Do I need to reconnect if credentials change?​

Yes. Any time you change Exely credentials:

  1. Update the attributes in Newo
  2. Click Save
  3. Click Publish All
  4. Run at least one live test

Does this integration support multiple properties?​

Not in the current implementation. One project uses one exely_property_id. For multiple properties, create separate Newo projects.

What currency are prices displayed in?​

Prices are returned in the property's configured currency in Exely (e.g., GEL for Georgia, EUR for Europe, USD for international properties).

Does Exely send webhooks for booking changes?​

No. Exely does not support webhooks. Booking status is checked on-demand during conversations.

10. Manager Rollout Summary​

This integration is a good fit if you want the AI to handle:

  • room availability inquiries
  • hotel reservation creation
  • reservation changes and cancellations
  • booking lookup by confirmation number

This integration is not currently positioned as:

  • a CRM replacement
  • a payment processing system
  • a guest recognition system based only on caller ID

Recommended rollout order:

  1. Enable API connections in Exely extranet
  2. Generate API credentials
  3. Configure in Newo and publish
  4. Test availability
  5. Test booking
  6. Test lookup, cancellation, modification
  7. Review Canvas scenarios
  8. Go live on one hotel first

Go-Live Recommendation​

For the cleanest launch:

  • start with one property only
  • keep all core features enabled unless there is a clear business reason not to
  • run at least one real booking and one real lookup before launch day
  • have a fallback human process ready for the first live shift
