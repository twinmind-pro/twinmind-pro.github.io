Altegio Integration Guide

What This Integration Does​

The Newo AI Agent integrates with Altegio so the agent can act as an appointment assistant for businesses that manage booking in Altegio.

Once connected, the AI can:

  • load Altegio services and staff into prompt context
  • check public Online Booking availability
  • create online bookings
  • cancel or reschedule Newo-created bookings that have a stored record_hash
  • search existing CRM appointments by phone number when B2B user authorization is configured
  • publish Altegio booking and appointment-management scenarios into the Newo Canvas

Before You Start​

You need:

  • an Altegio partner token
  • the Altegio location ID for the business branch served by this Newo project
  • service and staff IDs, or confidence that the service/staff cache can resolve customer requests
  • for CRM appointment lookup: an Altegio user token, or a user login/password that can obtain one through /auth
  • the user must have permission to read clients and appointment records for the selected location

Public booking uses:

Authorization: Bearer <partner_token>

B2B client and appointment lookup uses:

Authorization: Bearer <partner_token>, User <user_token>

Configuration​

Open Builder -> Settings -> Module - Altegio and configure the visible fields.

SettingRequiredWhat it does
altegio_partner_tokenYesAuthorizes public booking API calls and the partner part of B2B calls.
altegio_location_idYesLocation used for services, staff, availability, booking, and appointment lookup.
altegio_user_tokenFor B2B lookupEnables direct CRM client and appointment search.
altegio_user_loginOptionalUsed only when Newo must call /auth to obtain a user token.
altegio_user_passwordOptionalPassword paired with User Login for /auth. Prefer User Token when available.
altegio_default_service_idOptionalFallback service when the caller does not name a service.
altegio_default_staff_idOptionalFallback staff member for availability searches.
altegio_feature_availability_enabledYesTurns live slot lookup on or off.
altegio_feature_booking_enabledYesTurns booking creation on or off.
altegio_feature_cancellation_enabledYesTurns cancellation of Newo-created bookings on or off.
altegio_feature_reschedule_enabledYesTurns rescheduling of Newo-created bookings on or off.
altegio_feature_existing_appointments_enabledYesTurns phone-based existing appointment lookup on or off.
altegio_setup_scenariosOptionalPublishes default Altegio Canvas scenarios during setup.

Hidden system fields are auto-managed by the integration: altegio_base_url, altegio_booking_type, cache fields, lookup date-window fields, and setup persona ID.

Appointment Search By Phone​

The B2B lookup flow is:

  1. Extract or receive the caller phone number.
  2. If altegio_user_token is empty and login/password exist, call POST /auth.
  3. Call POST /company/{location_id}/clients/search with quick_search set to the phone number.
  4. Take the matched client ID.
  5. Call GET /records/{location_id}?client_id=...&start_date=...&end_date=...&with_deleted=0.
  6. Store the result in the prompt section ExistingBookings and persona field altegio_b2b_appointments.

Newo-created bookings remain in the normal bookings persona field. B2B records are kept separately because they often do not include the public record_hash required for public cancellation.

Supported API Coverage​

Closed in this integration:

  • GET /book_services/{location_id}
  • GET /book_staff/{location_id}
  • GET /book_times/{location_id}/{team_member_id}/{date}
  • POST /book_check/{location_id}
  • POST /book_record/{location_id}
  • DELETE /user/records/{record_id}/{record_hash}
  • PUT /book_record/{location_id}/{record_id}
  • POST /auth
  • POST /company/{location_id}/clients/search
  • GET /records/{location_id}

Not covered yet:

  • admin-side B2B booking creation through POST /records/{location_id}
  • B2B cancellation/reschedule of legacy CRM records without record_hash
  • webhooks for appointment/client changes
  • v2 appointment endpoints
  • marketplace lifecycle callbacks
  • VoIP call lookup

Verification Checklist​

After publishing:

  • confirm altegio_connector exists and is started
  • confirm Module - Altegio settings are saved
  • start a test conversation and ask for available appointment times
  • create one test booking and verify it appears in Altegio
  • ask to find appointments for the same phone number
  • confirm B2B lookup returns CRM appointments when User Token is configured
  • confirm the AI offers human handoff instead of inventing data when B2B credentials are missing
