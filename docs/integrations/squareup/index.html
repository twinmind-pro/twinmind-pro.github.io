Square Integration

Connects Square Appointments with the Newo AI agent so callers and chat visitors can ask about services, book, and cancel — entirely by voice or chat, no receptionist required.

What the AI can do​

  • Answer "do you offer X?" questions. Instead of guessing from a hand-written list, the agent searches the business's real Square catalog for any service the customer asks about, including variations, prices, and duration.
  • Check live availability. Open time slots are pulled live from Square Bookings (not a best guess).
  • Book the appointment. Once the customer picks a slot and gives their details, the agent creates the booking in Square — a receptionist sees it immediately.
  • Cancel an appointment. If the customer changes their mind, the agent looks up their booking and cancels it in Square.
  • Route multi-service requests. If a customer wants two or more services in one appointment, the agent politely hands off to a human manager instead of guessing which service comes first.

Service catalog — how it syncs to the agent​

Square is the single source of truth for services. The agent does not need the Products & Services list on the canvas to be kept up to date — it looks into Square directly.

When it happensWhat the agent does
First publishPulls the full Square catalog (items with product type Appointments Service) for the selected location, stores each variation in the agent's knowledge base with name, description, price, and duration.
Every re-publishRe-fetches the catalog, deletes the old snapshot from the knowledge base, writes the fresh one. No manual clean-up required.
New service added in SquareClick Publish All in Newo — the new service appears to the agent on the next customer turn.
Service renamed or re-priced in SquareSame — publish and the agent uses the new wording and price.
Service deleted in SquareSame — publish and the agent stops offering it.

What the agent actually knows about each service​

For every bookable variation the agent stores:

  • Service name (the parent item name, e.g. Eyebrow Threading).
  • Variation name (e.g. Regular, Yearly Colour Refresh).
  • Description from the Square catalog (if any).
  • Price — the fixed amount with its currency (e.g. 45.00 USD), or "variable" for variable-price services so the agent never quotes a wrong number.
  • Duration in minutes — used to pick the right slot window.

This is what lets the consultation scenario answer "How much is microblading?" with a real number and "How long does a full leg wax take?" with a real duration.

Only bookable services reach the agent​

Variations marked as "not available for booking" in Square, or restricted to a different location, are filtered out at sync time. Dispatchers don't need to configure anything extra — the agent will only ever offer and book what Square itself considers bookable right now.

Scenarios​

The integration ships with three editable scenarios on the canvas. Operators can tweak them in Newo; the agent picks up the changes on the next publish.

Scenario 7 — Service Consultation in Square​

Runs when the customer is asking about a service without yet asking to book. Rough flow:

  1. Identify the specific service the customer is asking about ("microblading", "laser hair removal", "eyebrow threading"). If the question is generic ("what do you offer?") the agent asks for a specific service.
  2. Look up the catalog — agent says "Let me check our services for you, I will get back to you shortly."
  3. Present matches:
    • Found → confirm the service is offered, list variations if several exist, quote price and duration, invite the customer to book.
    • Not found → politely decline and invite the customer to describe what they need so the agent can suggest a related service.
  4. If the customer wants to book, hand off to the Scheduling Appointment in Square via Agent scenario; otherwise wrap up.

Triggered by intent "[T] Service Consultation in Square".

Scenario 5 — Scheduling Appointment in Square via Agent​

Runs when the customer wants a new appointment. Rough flow:

  1. Ask what service is needed.
  2. Check live slots — agent says "Let me check available time slots for you, I will get back to you shortly." Real slots arrive with team members pre-matched.
  3. Customer picks a slot → confirm.
  4. Collect full name → phone → email.
  5. Submit the booking — agent says "Give me a moment, I will book your appointment right now." → booking confirmed in Square.
  6. If the customer asks for two or more services at any point, the agent transfers to a human manager instead of guessing which service goes first.

Triggered by intent "[L] Appointment in Square via Agent".

Scenario 6 — Canceling Appointment in Square via Agent​

Runs when the customer wants to cancel. Rough flow:

  1. Reconfirm the date/time of the booking, ask "Are you sure?".
  2. Customer confirms → agent says "Give me a moment, I will cancel your booking right now." → cancellation confirmed in Square.

Triggered by intent "[T] Cancellation in Square via Agent".

Each scenario's exact code-phrase is required — it's the trigger that signals the agent to actually call Square (vs. just talk about it). If you rewrite a scenario, keep those phrases intact.

Example dialogs​

Example 1 — "Do you offer X?"​

Customer: Hi, do you offer Hollywood wax? Agent: Let me check our services for you, I will get back to you shortly. (2–5 seconds — agent looks up the catalog) Agent: Yes, we do offer Hollywood wax! The regular Hollywood wax is 45.00 USD and takes 30 minutes. Would you like to book an appointment?

Example 2 — Asking for options​

Customer: Tell me about microblading, what are my options? Agent: Let me check our services for you, I will get back to you shortly. Agent: Yes, we offer microblading. We have Microblading for 300.00 USD, 90 minutes, and Microblading Yearly Colour Refresh for 180.00 USD, also 90 minutes. Which one would you like?

Example 3 — Service not in catalog​

Customer: Do you offer dragon breath therapy? Agent: Let me check our services for you, I will get back to you shortly. Agent: I'm sorry, we don't currently offer dragon breath therapy. If you'd like to describe what you need, I can try to suggest a related service.

Example 4 — Consultation then booking​

Customer: Do you do eyebrow threading? Agent: Let me check our services for you, I will get back to you shortly. Agent: Yes, we offer Eyebrow Threading for 15.00 USD, 10 minutes. Would you like to book? Customer: Yes please, tomorrow afternoon. Agent: Let me check available time slots for you, I will get back to you shortly. Agent: I have Friday April 24th at 2:00 PM, 2:30 PM, or 3:00 PM. Which works? Customer: 2:30 PM. Agent: May I have your full name? Customer: Sofia Ricci. Agent: And a phone number? Customer: +14155550132. Agent: And an email? Customer: sofia.ricci@example.com. Agent: Give me a moment, I will book your appointment right now. Agent: You're all set, Sofia — your Eyebrow Threading appointment is confirmed for Friday, April 24 at 2:30 PM.

What lands in Square:

  • New customer Sofia Ricci (or her existing profile if a phone match is found).
  • New appointment for Eyebrow Threading — Regular on the selected team member.

Example 5 — Multi-service → manager​

Customer: I'd like to book microblading and lip filler in the same visit. Agent: For appointments with multiple services, I'll need to connect you with our manager so they can arrange everything together. One moment… (Transfer to a human during working hours, or message-to-manager after hours.)

Example 6 — Policy question (not a service lookup)​

Customer: Can I come drunk to my appointment? Agent: We recommend avoiding alcohol for 24 hours before and after your treatment.

Policy questions are answered from the Business Context on the canvas, not from the catalog — the consultation tool is only fired for service lookups.

Setup​

A step-by-step walkthrough with screenshots is in Square Onboarding Guide (PDF). The section below is the short version.

1. Prepare your Square account​

Before connecting to Newo, set up the basics in Square itself.

1a. Add a location​

Open Square Dashboard → Locations and make sure at least one location exists.

Set the location's address, timezone, and business hours — the agent uses the location timezone for every slot and booking.

1b. Turn on Square Appointments for the location​

Open Square Appointments → Settings and enable Appointments for the location you just configured.

Without this step, Square returns an empty catalog and the agent has nothing to offer.

1c. Add at least one team member who can take bookings​

Open Square Appointments → Staff and add team members. Each team member must be set as available for booking for the services they deliver.

1d. Define services​

Open Square Appointments → Services and add every service the agent should be able to answer about and book.

For each service:

  • Price and duration are required — those values travel into the agent's knowledge base and are what the agent quotes to the customer.
  • Variations (e.g. Short hair / Medium hair / Long hair, 1 session / 6 sessions) are each their own bookable unit with their own price and duration.
  • Tick Available for booking on every variation the agent should offer.
  • Use VARIABLE_PRICING only when you really mean it — the agent will then say "price is variable" instead of quoting a number.

1e. Confirm availability​

Open Square Appointments → Calendar / Availability and make sure each team member has working hours set up.

If a team member has no working hours, Square won't return any slots and the agent will say "Sorry, we have no availability".

2. Get your Square credentials​

Option A — Personal Access Token (recommended)​

Simplest path. Works for any Square merchant.

  1. Go to https://developer.squareup.com/apps.
  2. Create a new application, or use an existing one.
  3. Open the app → Credentials → Production tab.
  4. Copy the Personal Access Token.

Option B — OAuth 2.0​

Use this if you need scoped access or are connecting on behalf of another seller.

  1. In the same Developer Dashboard app, copy the Application ID (Client ID) and Application Secret.
  2. During setup (see next section) you'll be redirected to Square to authorize the app and will paste the resulting code back into Newo.

3. Configure in Newo​

  1. Open the project and find the Module - Square Integration attribute group.
  2. Choose auth method in square_auth_type (Token or OAuth).
  3. For Token: paste the token into square_personal_token. Hit Publish All.
  4. For OAuth: fill in square_client_id and square_secret_id, follow the Square authorize link, paste the returned code into square_oauth_code, then hit Publish All.
  5. After the first successful auth, select the location in the square_locations dropdown (auto-populated) and hit Publish All again. The agent filters services and availability by this location.
  6. (Optional) review the feature toggles:
SettingPurposeDefault
Availability FeatureTurns live slot lookup on/offon
Booking FeatureTurns booking creation on/offon
Cancellation FeatureTurns cancellation on/offon
Service Consultation FeatureLets the agent answer "do you offer X?" and "how much is Y?" from the Square catalogon
Auto-Setup Canvas ScenariosPublishes the default consultation/booking/cancel scenarios on first installon

4. Hit Publish All​

On publish, the agent will:

  • Create the Square connector.
  • Register the four custom tools (consultation, availability, booking, cancellation).
  • Pull the Square catalog for the selected location and load it into the agent's knowledge base.
  • Add the consultation / booking / cancellation scenarios and intents to the canvas (only if no conflicting items are already there — existing customizations are preserved).

Place a test chat message or call to confirm.

Customizing the agent​

The scenario bodies on canvas are fully editable. Common tweaks:

  • Ask for more info during booking. Add steps like "ask for the allergy history" or "ask for preferred staff". Whatever the agent collects is available to the scenario from then on.
  • Change the consultation tone. Scenario 7 can be rewritten — just keep the code-phrase "Let me check our services for you, I will get back to you shortly." intact so the lookup still fires.
  • Restore defaults. If you break a scenario, delete it from the canvas and republish — the original is restored from the integration's library.

Frequently asked questions​

Q. Does the agent ever charge the customer? No. It only books and cancels. Payment is handled inside Square by staff as normal.

Q. What happens if two customers try to book the same slot? Square serializes the bookings. The second request comes back with a "slot no longer available" error; the agent apologizes and offers the remaining slots.

Q. Can the caller reschedule in one step? Not directly — the agent cancels the old booking and starts a new booking flow.

Q. The customer asked about a service we stopped offering last week. What does the agent say? After the next Publish All, the old service is gone from the knowledge base — the consultation scenario will politely tell the customer it's not offered and invite them to suggest a related service.

Q. We just added a new service in Square. When does the agent see it? Hit Publish All in Newo. On the next customer turn the agent already knows about the new service with its price and duration.

Q. Does the agent answer policy questions (hours, parking, allergy disclaimers…)? Yes — those come from the Business Context you configure on the canvas, not from the Square catalog. The consultation tool only fires on service questions.

Q. Can I connect multiple Square locations at once? One location per integration instance. The location is chosen during setup from an auto-populated list.

Q. What happens if the token expires or is revoked? Token mode: tokens don't expire but can be revoked. Generate a new one and update square_personal_token. OAuth mode: the integration auto-refreshes on 401 using the refresh token — no manual action needed.

Q. Can I turn off the consultation tool if I prefer the old "only books, doesn't answer questions" behaviour? Yes — set Service Consultation Feature to off and republish. The agent will fall back to the canvas Products and Services block for service questions.

Limitations​

  • Reschedule endpoint is not wired (Square API shape differs per region). Workaround: cancel + rebook.
  • Multi-service bookings are intentionally routed to a human manager. The agent does not attempt to pack two services into one appointment.
  • Historical data — the agent only sees services and bookings that exist in Square at or after the first publish.
  • One location per instance — to serve two locations, deploy two Newo projects.

Changelog​

v1.2.0 — Service Consultation tool + live pricing + better extraction​

  • New Service Consultation tool and scenario. Customers can now ask "do you offer X?", "how much is Y?", "tell me about Z" and get an accurate answer pulled live from the Square catalog — including price and duration — instead of being told the service isn't available because it wasn't listed on the canvas.
  • Price and duration are now part of the agent's knowledge. Every bookable variation stored in the knowledge base carries its fixed price (or a "variable" marker) and duration in minutes, so the agent can quote both confidently.
  • Availability extraction stabilised. The agent now uses a strict JSON schema for pulling service name / date / time from the conversation and a calendar snippet to resolve "tomorrow" / "next Monday" correctly. Previous versions occasionally hallucinated a past year.
  • Booking extraction uses the same calendar-aware JSON schema.
  • Catalog lookup made more forgiving. The fuzzy match now searches across service name and summary with a permissive score threshold, so "microbladding" (typo) resolves to Microblading and the correct variation ID ends up in the Square request.
  • Four new hidden advanced settings — JSON schemas and instructions for availability, booking, and consultation extraction — so operators can fine-tune the agent's extraction behaviour without code changes.
