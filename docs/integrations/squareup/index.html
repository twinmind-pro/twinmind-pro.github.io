Square Integration

1. Overview​

1.1 Description​

Square Integration connects Square (appointment scheduling and POS platform) with Newo Platform.

It enables:

  • Checking real-time appointment availability from Square Bookings
  • Booking appointments with automatic customer creation in Square
  • Cancelling existing appointments
  • Synchronizing service catalogs (services, variations, pricing) from Square
  • LLM-powered service matching from natural language requests
  • Team member assignment based on availability per slot
  • Two authentication methods: Personal Access Token (recommended) and OAuth 2.0
  • Auto-configuring conversation canvas with booking and cancellation scenarios

This integration is designed for service businesses (salons, spas, healthcare, etc.) who need automated appointment scheduling through a conversational AI agent (voice or chat).

1.2 Use Cases​

  • When a client asks about availability → Query Square Bookings for open slots with matched service
  • When a client requests a specific service → LLM matches natural language to service catalog variations
  • When a client wants to book → Find or create customer, assign team member, create booking
  • When a client wants to cancel → Cancel booking via Square API with idempotency
  • When a client requests two or more services at once → Transfer to manager

2. Requirements​

Before installation:

  • Active Square account with Bookings enabled
  • At least one location configured in Square
  • Services defined in Square Catalog (product type: APPOINTMENTS_SERVICE)
  • One of:
    • Personal Access Token from Square Developer Dashboard (recommended, simpler)
    • Square OAuth App with Client ID and Secret

3. How to Setup​

3.1 Option A — Personal Access Token (Recommended)​

The simplest way to connect. No OAuth flow required.

  1. Go to https://developer.squareup.com/apps
  2. Create a new application (or use an existing one)
  3. Go to Credentials → Production tab
  4. Copy the Application ID and Personal Access Token
  5. In Newo platform, set:
    • square_auth_type = Token
    • square_personal_token = your copied token
  6. Click Publish All

For a detailed guide with screenshots, see the Square Onboarding Guide (PDF).

3.2 Option B — OAuth 2.0​

Use this if you need scoped access or are connecting on behalf of another seller.

  1. In Newo platform, set square_auth_type = OAuth
  2. Set square_client_id and square_secret_id
  3. Click the Square authorization link provided by the platform
  4. Log in to your Square account and authorize the app
  5. Copy the authorization code from the redirect page
  6. Paste the code into the square_oauth_code attribute
  7. Click Publish All

OAuth scopes requested: ITEMS_READ, APPOINTMENTS_READ, APPOINTMENTS_WRITE, APPOINTMENTS_ALL_READ, APPOINTMENTS_ALL_WRITE, EMPLOYEES_READ, CUSTOMERS_WRITE, CUSTOMERS_READ, MERCHANT_PROFILE_READ

4. How to Use​

4.1 How the Integration Works​

The integration uses Square Bookings API for scheduling, Catalog API for services, and Customers API for contact management. Availability is computed via the bookings availability search, and services are matched from the synced catalog using LLM.

  • A Trigger is a system event that starts a flow
  • An Action is the API operation performed in Square

Available Triggers​

TriggerEventDescription
Check Availabilityget_available_slotsWhen the agent needs to look up open slots
Book Appointmentbook_slotWhen the agent confirms a booking
Cancel Appointmentconvo_agent_cancel_bookingWhen the agent processes a cancellation
Catalog Syncsession_startedAuto-sync service catalog on conversation start
Setupproject_publish_finishInitializes integration on deploy
Canvas Setupgm_workflow_builder_canvas_builtAuto-adds booking/cancellation scenarios

Available Actions​

  • Search Availability — Find open slots by service and date range (POST /v2/bookings/availability/search)
  • Search Customer — Find customer by phone (POST /v2/customers/search)
  • Create Customer — New customer record (POST /v2/customers)
  • Create Booking — Schedule appointment with team member (POST /v2/bookings)
  • Cancel Booking — Cancel appointment (POST /v2/bookings/{id}/cancel)
  • Search Catalog — Sync service variations (POST /v2/catalog/search)
  • Get Locations — List merchant locations (GET /v2/locations)

4.4 Canvas Scenarios​

When square_setup_scenarios is True (default), the CanvasSetupFlow automatically adds these to the conversation canvas on publish:

Scenarios:

  • "Scheduling Appointment via Agent" (square_booking_scenario) — full booking flow with multi-service guard (2+ services → transfer to manager)
  • "Canceling Appointment via Agent" (cancellation_via_agent_scenario) — guided cancellation with confirmation

Intent Types:

  • "[T] Appointment Request via Agent" (square_booking_intent)
  • "[T] Cancellation via Agent" (cancellation_via_agent_intent)

These scenarios can be customized on the canvas after initial setup — modified scenarios are not overwritten on subsequent publishes.

4.5 Where to Test​

Testing can be done through the Newo conversation interface (chat or voice). Each flow has dedicated webhook test endpoints for connectivity validation.

4.6 Testing and Verification​

To test the integration:​

  1. Setup: Publish the project and verify token is set (Token mode) or OAuth exchange completed
  2. Catalog: Start a conversation — verify square_catalog is populated with service variations
  3. Availability: Ask "Any openings for [service] tomorrow?" — verify slots returned with team members
  4. Booking (single service): Complete a booking — verify appointment created in Square
  5. Booking (multi-service): Request two services — verify agent transfers to manager
  6. Cancellation: Cancel a booking — verify appointment deleted from Square

If no action occurs:​

  • Ensure authentication is configured:
    • Token mode: square_auth_type = Token and square_personal_token is set
    • OAuth mode: square_client_id, square_secret_id are set and square_access_token was obtained
  • Confirm square_locations is selected (should be auto-populated as enum)
  • Check square_catalog is not empty (services must exist in Square as APPOINTMENTS_SERVICE)
  • Verify feature flags are enabled (square_enable_slot_check, square_enable_booking, square_enable_cancellation)
  • Re-publish the project if credentials were changed

Note: This integration creates real appointments in Square. Bookings and cancellations are reflected in the live Square system.

5. FAQ​

Q: What is the difference between Token and OAuth authentication? A: Token (Personal Access Token) is simpler — the business owner copies a token from their Square Developer Dashboard. It gives full access to their account without scopes. Best for single-merchant setups. OAuth uses the standard authorization flow with scoped permissions. Best for multi-merchant or marketplace scenarios.

Q: How does the agent match services from natural language? A: The integration uses Gemini 2.5 Flash (temperature=0.2) to match the user's service request (e.g., "manicure") against the synced square_catalog of service variations. It returns the matching service_variation_id used for the availability search and booking.

Q: What happens when a client requests two or more services? A: The agent detects multiple services and does NOT attempt to book. Instead, it explains that multi-service appointments require manager assistance and initiates a transfer (during working hours) or relays a message to the manager (after hours).

Q: How are team members assigned to appointments? A: During the availability search, Square returns which team members are available for each slot. The integration stores this mapping (square_free_teamembers) in the persona. When booking, the correct team member is looked up for the selected time.

Q: Why are idempotency keys used? A: All write operations (customer creation, booking, cancellation) use UUID idempotency keys to prevent duplicate operations if requests are retried. This is a Square API best practice.

Q: When is the service catalog synced? A: The catalog syncs automatically on session_started (every new conversation). It filters for items with product_type: APPOINTMENTS_SERVICE that are available at the selected location.

Q: Can I connect multiple Square locations? A: The integration supports one location per instance. During setup, available locations are fetched and presented as an enum (square_locations) in the format "id | name | timezone". Select the desired location after initial auth.

Q: What happens if the token expires or is invalid? A: Token mode: The agent returns an error (Personal Access Tokens don't expire, but can be revoked). The business owner needs to generate a new token. OAuth mode: The integration detects HTTP 401 responses and automatically refreshes the token via /oauth/v2/refresh. Failed requests are retried.

Q: What customer information is needed for booking? A: The customer's phone number is required for lookup/creation. Name (first/last) is used when creating a new customer. The integration splits full_name into first and last names automatically.

Q: Can I disable the auto-added canvas scenarios? A: Yes, set square_setup_scenarios to False before publishing. Existing scenarios on the canvas will not be removed.
