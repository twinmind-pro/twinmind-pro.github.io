Monday.com Integration

1. Overview​

1.1 Description​

Monday.com Integration connects Monday.com (work management platform) with Newo Platform.

It enables:

  • Checking appointment availability based on existing meetings in Monday boards
  • Creating new bookings as meeting items in Activities board with contact linking
  • Cancelling appointments by archiving meeting items
  • Detecting existing customers by phone and retrieving their upcoming bookings
  • Managing two interconnected boards: Activities (meetings) and Contacts (customers)
  • Optional silent availability preload on conversation start

This integration is designed for service businesses and teams who use Monday.com boards for scheduling and need automated appointment management through a conversational AI agent (voice or chat).

1.2 Use Cases​

  • When a client asks about availability → Query Activities board for meetings, compute free slots
  • When a client wants to book → Find or create contact in Contacts board, create meeting in Activities board
  • When a client wants to cancel → Archive the meeting item in Activities board
  • When a caller's phone is identified → Look up contact and retrieve upcoming bookings
  • When a conversation starts → Optionally preload nearest available slots silently

2. Requirements​

Before installation:

  • Active Monday.com account
  • Monday.com OAuth App with Client ID and Client Secret
  • Two pre-configured boards:
    • Activities board — for meetings/events (must have columns: activity_type, activity_status, activity_start_time, activity_end_time, activity_item)
    • Contacts board — for customer records (must have column: contact_phone)

3. How to Setup​

3.1 Step 1 — Authorize in Monday.com​

  1. Create an OAuth app in Monday.com developer portal
  2. In Newo platform, set monday_client_id and monday_client_secret
  3. Obtain the OAuth authorization code
  4. Paste the code into the monday_oauth_code attribute

3.2 Step 2 — Connect in Platform​

  1. Open Newo → Projects
  2. Set the following attributes in Builder / Attributes:
AttributeRequiredDescription
monday_client_id✅OAuth Client ID
monday_client_secret✅OAuth Client Secret
monday_oauth_code✅One-time OAuth authorization code
activities_id✅Board ID for meetings/activities
contacts_id✅Board ID for contacts
monday_base_api_url❌GraphQL API endpoint (default: https://api.monday.com/v2)
monday_timezone❌Booking timezone (default: project timezone)
monday_duration❌Slot duration in minutes (default: 30)
monday_show_for_days❌Days ahead for availability (default: 1)
monday_check_availability_on_conversation_start❌Auto-check availability on start (default: False)
monday_slots_available❌Enable availability checking (default: True)
monday_booking_available❌Enable booking capability (default: True)
monday_cancelation_available❌Enable cancellation capability (default: True)
  1. Click Save + Publish All
  2. On publish, the SetupFlow automatically:
    • Exchanges the OAuth code for an access token
    • Queries the Activities board for column structure (auto-detects date and notes columns)
    • Discovers the group ID for activity items
    • Injects booking and availability payload schemas

4. How to Use​

4.1 How the Integration Works​

The integration uses Monday.com GraphQL API to manage two boards: Activities (meetings) and Contacts (customers). Availability is computed by querying existing meetings and finding gaps. Bookings create new items linked to contacts.

  • A Trigger is a system event that starts a flow
  • An Action is the GraphQL operation performed in Monday.com

Available Triggers​

Available Actions​

  • Query Activities — Fetch meetings with filters (GraphQL items_page on Activities board)
  • Create Meeting — New meeting item with dates and contact link (GraphQL create_item)
  • Archive Meeting — Cancel appointment (GraphQL archive_item)
  • Query Contacts — Search by phone (GraphQL items_page on Contacts board)
  • Create Contact — New contact with phone (GraphQL create_item)

4.3 Where to Test​

Testing can be done through the Newo conversation interface (chat or voice channel).

4.4 Testing and Verification​

To test the integration:​

  1. Setup: Publish and verify OAuth token + board structure discovery
  2. Existing Client: Call from a known phone — verify customer info injected into prompt
  3. Availability: Ask about open slots — verify free times computed from Activities board
  4. Booking: Complete a booking — verify meeting item created in Activities board linked to contact
  5. Cancellation: Cancel a booking — verify item archived in Monday.com

If no action occurs:​

  • Ensure OAuth credentials are set (monday_client_id, monday_client_secret)
  • Verify monday_access_token was obtained
  • Confirm activities_id and contacts_id are set to valid board IDs
  • Check that the Activities board has required columns (activity_type, activity_status, activity_start_time, activity_end_time)
  • Verify feature flags are enabled
  • Re-publish the project if settings were changed

Note: This integration creates and archives real items in Monday.com boards.

5. FAQ​

Q: What board structure is required? A: Two boards are needed. The Activities board must have columns: activity_type (Meeting), activity_status (Open/Done), activity_start_time, activity_end_time, activity_item (relationship to Contacts). The Contacts board must have contact_phone column.

Q: How is availability computed? A: The integration queries the Activities board for meetings (type=Meeting, status≠Done) within the date range, then uses a sliding window algorithm to find gaps. Slot duration is configurable via monday_duration (default: 30 min).

Q: How does cancellation work? A: Cancellation uses Monday.com's archive_item mutation — it doesn't delete the item but archives it, making it recoverable. The booking_id is the Activities board item ID.

Q: Are date and notes columns auto-detected? A: Yes. During setup, the integration queries the Activities board columns and auto-detects date columns (by date_ ID prefix or "Date" title) and notes columns (by long_text_ ID prefix or "Notes" title). If not found, items are still created without these optional columns.

Q: Can I use existing Monday.com boards? A: Yes, as long as they have the required column structure. Set the board IDs in activities_id and contacts_id attributes.

Q: What happens with token refresh? A: The UtilsFlow handles HTTP 401 responses, but the token refresh implementation for Monday.com is currently a placeholder. If tokens expire, re-authorization may be needed.
