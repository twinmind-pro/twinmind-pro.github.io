FollowUpBoss Integration

1. Overview​

1.1 Description​

FollowUpBoss Integration connects FollowUpBoss (real estate CRM for agents and teams) with Newo Platform.

It enables:

  • Creating or updating leads in the CRM with auto-deduplication via POST /events
  • Looking up existing contacts by phone number to personalize conversations (lead stage, assigned agent)
  • Checking appointment availability by querying existing appointments and computing free slots
  • Scheduling appointments for showings, consultations, and meetings
  • Cancelling existing appointments
  • Automatically logging calls and conversation summary notes to contact records after the session ends

This integration is designed for real estate agents and team administrators who want to capture leads and manage appointments through an AI receptionist (voice or chat).

1.2 Use Cases​

  • When a caller provides their contact info as a new lead → Create lead via POST /events with source tracking
  • When a caller is an existing contact → Look up their profile, display lead stage and assigned agent in the conversation
  • When a caller asks about available times → Query existing appointments, compute free 30-min slots in 9-17 window, present options
  • When a caller books a time → Create appointment in FUB with person ID and assigned agent
  • When a caller cancels an appointment → Cancel via DELETE /appointments/{id}
  • When a session ends → Automatically log the AI call and attach a conversation summary note to the contact

2. Requirements​

Before installation:

  • Active FollowUpBoss account (trial or paid)
  • FollowUpBoss API Key (Admin → API)
  • X-System and X-System-Key registered at https://apps.followupboss.com/system-registration (one-time setup per integration)

3. How to Setup​

3.1 Step 1 — Obtain FollowUpBoss Credentials​

  1. Log in to your FollowUpBoss account
  2. Navigate to Admin → API to get your API Key
  3. Register your integration at https://apps.followupboss.com/system-registration to obtain:
    • X-System (system name, e.g., NewoAI)
    • X-System-Key (system-level API key)
  4. Store all credentials securely

3.2 Step 2 — Connect in Platform​

  1. Open Newo → Projects
  2. Import followupboss.zip to the project
  3. Set the following attributes in Builder / Attributes:
AttributeRequiredDescription
followupboss_api_key✅FollowUpBoss API key (Admin → API)
followupboss_x_system✅Registered system name (X-System header)
followupboss_x_system_key✅Registered system API key (X-System-Key header)
followupboss_base_url❌API base URL (default: https://api.followupboss.com/v1)
followupboss_default_event_source❌Lead source name (default: newo.ai)
followupboss_default_appointment_type_id❌Default appointment type ID (auto-populated from catalog)
followupboss_default_agent_user_id❌Default agent to assign (auto-populated from users catalog)
followupboss_override_agent_attributes❌Override superagent settings on setup (default: True)
followupboss_setup_scenarios❌Auto-add canvas scenarios (default: True)
followupboss_enable_post_call_log❌Log call to contact timeline after session (default: True)
followupboss_enable_post_call_note❌Add conversation note after session (default: True)
  1. Click Save + Publish All
  2. On publish, the SetupFlow automatically:
    • Creates HTTP connector (followupboss_connector)
    • Sets superagent booking attributes (booking_enabled, booking_type=project, booking_cancellation_enabled, etc.)
    • Fetches appointment types and users from FUB, caches them
    • Registers 2 custom tools (followupboss_lookup_contact, followupboss_create_lead)
    • Adds 5 canvas scenarios and 5 intent types (real estate workflows)

4. How to Use​

4.1 How the Integration Works​

The integration operates based on Triggers and Actions via the event-driven system.

  • A Trigger is a system event that starts a flow
  • An Action is the API operation performed in FollowUpBoss

Triggers​

Setupproject_publish_finishInitializes integration on deploy
Canvas Setupgm_workflow_builder_canvas_builtAdds custom intents/scenarios to canvas
Auto-Identify Callerdefine_user_phone_numberLooks up contact when phone is detected
Lookup Contact Toolfollowupboss_lookup_contactLLM-triggered contact lookup
Create Lead Toolfollowupboss_create_leadLLM-triggered lead creation
Check Availabilityget_available_slotsConvoAgent requests free time slots
Book Appointmentbook_slotConvoAgent creates appointment
Cancel Appointmentconvo_agent_cancel_bookingConvoAgent cancels existing appointment
Post-Call Actionscustom_session_endedFires on session end → log call + add note

Available Actions​

  • Search People — GET /people?phone={phone}&limit=1
  • Create/Update Lead — POST /events (auto-deduplicates)
  • Get Appointment Types — GET /appointmentTypes
  • Get Users — GET /users
  • Create Appointment — POST /appointments with invitees
  • List Appointments — GET /appointments?fromDate=...&toDate=... (for availability)
  • Cancel Appointment — DELETE /appointments/{id}
  • Log Call — POST /calls with personId, outcome, note
  • Add Note — POST /notes with personId, subject, body

6. Authentication​

FollowUpBoss uses two-layer authentication:

Layer 1 — System Identification (required for integrations)​

Every request includes:

  • X-System: Registered system name
  • X-System-Key: System API key

Layer 2 — User Authentication (Basic Auth)​

HTTP Basic Auth with API key as username, empty password:

Authorization: Basic base64(api_key:)

The integration includes a custom base64 library (libraries/base64/encode.nsl) because NSL Jinja2 doesn't expose Python's base64 module.

7. Canvas Setup​

On publish, CanvasSetupFlow adds 5 intent types and 5 scenarios to the canvas:

Intent Types​

  • followupboss_appointment_booking_intent — Schedule appointment
  • followupboss_lead_capture_intent — New lead
  • followupboss_existing_contact_lookup_intent — Look up existing contact
  • followupboss_property_inquiry_intent — Property question
  • followupboss_cancellation_intent — Cancellation request

Scenarios​

  • followupboss_appointment_booking_scenario — Booking flow with code-phrases for availability/booking
  • followupboss_lead_capture_scenario — Lead capture with "Let me save your information in our CRM now" code-phrase
  • followupboss_existing_contact_lookup_scenario — Contact identification
  • followupboss_property_inquiry_scenario — Property Q&A + lead capture fallback
  • followupboss_answering_questions_scenario — General Q&A
  • followupboss_cancellation_scenario — Cancellation with "I'm cancelling your booking now" code-phrase

Platform defaults are removed from canvas to avoid conflicts:

  • scenario_sales_appointment_via_calendar (Google Calendar tool)
  • scenario_home_service_appointment_request
  • scenario_common_schedule_appointment_agent
  • intent_type_appointment_scheduling, etc.

9. Rate Limits (FollowUpBoss API)​

Sliding 10-second window:

  • Global: 250 req/10s (with X-System-Key), 125 without
  • POST /events: unlimited
  • Notes: 10 req/10s
  • PUT /people: 25 req/10s

On HTTP 429, Retry-After header indicates wait time.

10. Known Limitations​

  • Google Calendar hijack — If customer has project_attributes_setting_calendar_event_integration=google_integration enabled, Google Calendar may intercept booking before FUB flow fires. SetupCanvasSkill deletes scenario_sales_appointment_via_calendar to prevent this.
  • Appointment reading — FUB API only returns appointments created IN FUB (not synced from Google Calendar) and belonging to the requesting API user.
  • Session timeout — Post-call flow fires on custom_session_ended, which is platform-driven (project_attributes_setting_conversation_session_timeout_unfinished_chat, default 300s).
